那么，你创建了一个 package.json 文件，可以它缺少一些人们通常期望的内容。
如果你输入 "npm install"，你会看见大致如下的内容：

    npm WARN package.json %ID% No description
    npm WARN package.json %ID% No repository field.
    npm WARN package.json %ID% No README data

在向世界分享你的工作成果之前，我们需要将其再完善一下，以便人们知道如何使用它。

首先，创建一个README.md文件，里面只有少量的文字。

然后，在package.json文件中添加名为 "repository" 字段，并制定一个人们可以访问代码的网址。

你能手动编辑你的package.json文件，或运行再次 "npm init"。

当你完成后，运行"$ADVENTURE_COMMAND verify"。
